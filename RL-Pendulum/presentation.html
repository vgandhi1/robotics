The Problem

PID/LQR is brittle math. RL is a learned policy.

Traditional inverted-pendulum control breaks when hardware ages, floors get slippery, or batteries drain. A domain-randomized RL policy handles non-linearity and noise natively.

PID / LQRRL Policy (this project)

02
Pipeline

Four phases: digital twin → policy → randomize → export

Phase 1 — Digital Twin Custom Gymnasium env, Euler-integrated 2-wheel dynamics, complementary-filter IMU model matched to MPU-6050 Phase 2 — Baseline (5M) PPO (Stable Baselines3), MLP 4 → [64,64] → 1 (tanh) Reward R = a(1-th^2) - b*act^2 - c|x| + d Phase 3 — Domain Rand (10M) mass +-20%, friction +-50%, IMU noise 1-20 mrad, latency 5-15ms, wheel slip 0-5%, action delay 0-2 steps Phase 4 — Export PyTorch FP32 -> ONNX FP32 -> ONNX INT8 -> TFLite INT8
